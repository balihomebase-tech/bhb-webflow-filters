# BHB Webflow Filters

A modular, production-ready filtering system for Bali Home Base property listings. Three independent filters (Rentals, Villas, Land) with shared utilities, core logic, and a redesigned architecture for maintainability.

- **Rentals**: Availability-aware rental property filter
- **Villas**: Ownership-based villa filter with dynamic price ranges
- **Land**: Land size and price-per-are calculations

## How It Works

### Filter Pipeline

1. **User selects filter criteria** (location, price, availability, ownership, etc.)
2. **`applyFilters()` runs** (from core/filter-engine.js)
3. **Each card is checked** via page-specific `passes()` function
4. **Results are updated** with pagination (load more)
5. **Map markers refresh** to show matching locations
6. **UI updates** (result count, empty state, buttons)

### Price Filtering Differences

**Rentals**: Uses static preset chips (< 50jt, 50jt–200jt, > 200jt, etc.)

**Villas**: Calculates dynamic price range from CMS data
- When ownership filter changes → price range recalculates
- Chips are hidden (no presets for dynamic ranges)
- If ownership = "Any" → uses all villa prices
- If ownership = "Freehold" → uses only freehold prices
- If ownership = "Leasehold" → uses only leasehold prices

**Land**: Uses static presets per are (< Rp8jt/are, Rp8jt–15jt/are, > Rp15jt/are, etc.)

## Configuration

### Area Groupings

All three pages use the same area structure:

| Area | Locations |
|------|-----------|
| **Canggu area** | Canggu, Pererenan, Seseh, Cemagi, Buduk, Kaba Kaba, Kaba-Kaba, Cepaka, Tumbak Bayuh, Buwit, Dalung |
| **Uluwatu area** | Bingin, Uluwatu, Uluwatu Center, Ungasan |
| **Ubud area** | Ubud, Ubud Center |
| **Tabanan area** | Kedungu, Nyanyi, Pandak Gede, Nyambu, Tanah Lot |

To customize, edit `js/configs/{page}-config.js`:

```javascript
var AREA_RULES = [
  {
    id: "canggu-area",
    label: "Canggu area",
    keys: ["canggu", "pererenan", "seseh", "cemagi", "buduk", ...]
  },
  // ... more areas
];

var LOC_COORDS = {
  canggu: [115.1365, -8.65062],
  pererenan: [115.12346, -8.64904],
  // ... more locations with [longitude, latitude]
};
```

### Price Presets (Rentals & Land Only)

Edit `CHIP_PRESETS` in `js/configs/{page}-config.js`:

```javascript
var CHIP_PRESETS = {
  IDR: [
    { label: "< 50jt", min: 0, max: 50000000 },
    { label: "50jt – 200jt", min: 50000000, max: 200000000 },
    { label: "> 200jt", min: 200000000, max: null }
  ],
  USD: [
    { label: "< $3k", min: 0, max: 3000 },
    { label: "$3k – $12k", min: 3000, max: 12000 },
    { label: "> $12k", min: 12000, max: null }
  ],
  EUR: [ /* ... */ ]
};
```

> **Note**: Villas ignore `CHIP_PRESETS`. It uses dynamic price ranges calculated from CMS data based on the ownership filter.

### Grid & Card Settings

Edit `CFG` in `js/configs/{page}-config.js`:

```javascript
var CFG = {
  GRID_ID: "rentals-wrapper",    // ID of grid container
  CARD_SEL: ".w-dyn-item",       // Selector for each card
  STEP: 9                         // Cards to load per "Load More" click
};
```

### Map Settings

Edit these in `js/configs/{page}-config.js`:

```javascript
var MAPTILER_KEY = "your-api-key-here";
var MAP_STYLE = "019c8e23-ebd1-7221-bd5f-20ae2dca2ab6";
var PIN_URL = "https://cdn.prod.website-files.com/.../Icon.svg";

```

The filters support IDR, USD, and EUR with real-time conversion.

```javascript
// Call these from your global currency switcher
window.debugCurrency.convertAmount(amount, fromCurrency, toCurrency);
```

## Development & Updating

### Local Development

1. Clone/fork the repository
2. Edit files in your local `js/` directory
3. Test changes in Webflow (use local URLs or staging CDN)
4. Commit changes to `main`
5. Create release tags for stable versions

### Publishing Updates

```bash
# Make your changes and commit
git add .
git commit -m "feat: description of changes"

# Create a release tag
git tag v1.2.3
git push origin main --tags
```

Update Webflow script tags to use the new tag:
```html
<!-- Instead of @main, use specific version -->
<script src="https://cdn.jsdelivr.net/gh/USERNAME/bhb-webflow-filters@v1.2.3/js/pages/rentals.js"></script>
```

jsDelivr will cache the @main version aggressively, so versioned tags are recommended for production.

## Troubleshooting

**Filters not loading?**
- Check console for errors
- Verify all script tags are in correct order
- Confirm grid ID matches `CFG.GRID_ID`

**Price range incorrect?**
- Check `data-currency` attributes on cards
- For Villas: ensure `data-available` is set (Leasehold/Freehold)
- Verify prices are numeric values

**Map not showing?**
- Ensure `id="bhbMap"` exists in DOM
- Check MapTiler key is valid
- Confirm location has coordinates in `LOC_COORDS`

**Currency conversion not working?**
- Ensure `window.debugCurrency` is defined
- Call `window.debugCurrency.setCurrency("IDR")` from your switcher

**Load more button not working?**
- Rentals: Check `id="load-more"` exists
- Villas: Check `.villas-load-more` class exists
- Land: Check `id="load-more"` exists

## Technical Details

### Script Dependencies

```
Page Script
  ├── Core Filter Engine
  ├── Utils → Currency
  ├── Utils → Map
  └── Config (AREA_RULES, LOC_COORDS, CHIP_PRESETS, etc.)
```

### Global State

Each page maintains a `state` object:
```javascript
{
  ownership: "Any",        // Rentals: ignored, Villas/Land: Leasehold/Freehold
  bedrooms: [],            // Rentals/Villas: [1, 2, 3, ...]
  lease: "Any",            // Villas/Land: lease duration
  locations: [],           // All: selected location names
  currency: "IDR",         // All: current currency
  priceMin: null,          // All: minimum price filter
  priceMax: null,          // All: maximum price filter
  keyword: ""              // All: search query
}
```

### Filter Order

Filters are applied with AND logic:
1. Location (if selected)
2. Ownership (if selected)
3. Bedrooms (if selected)
4. Lease duration (if selected)
5. Price range
6. Keyword search
7. Page-specific filters (availability, land size, etc.)
